    Foreign key, Khóa ngoại là gì?

    • Nguyễn Mạnh Tùng
    • /
    • 12/05/2018
    • 19/05/2018

    Khóa ngoại (FOREIGN KEY), dùng để tạo quan hệ giữa các bảng. Khóa ngoại của bảng này sẽ dẫn tới khóa chính của bảng khác. Hãy nhìn vào ví dụ sau.

    Bảng Customers trong database NorthWind:

    Bảng Orders trong database NorthWind:

    Ta thấy rằng, cột CustomerID trong bảng Orders sẽ dẫn đến cột CustomerId trong bảng Customers.

    Cột CustomerId trong bảng Customers là PRIMARY KEY của bảng Customers.

    Cột CustomerId trong bảng Orders là khóa ngoại (FOREIGN KEY) của bảng Orders.

    Hai bảng này liên kết với nhau qua cặp khóa chính và khóa ngoại. Với mối quan hệ một nhiều, một Customer sẽ có nhiều Order.

    Ràng buộc khóa ngoại dùng để chống lại những hành động có thể dẫn đến việc thêm vào những dữ liệu sai.

    Ví dụ: Ở bảng Customers không có Customer nào có CustomerId = ‘AFKLOL’, tuy nhiên bạn lại insert dữ liệu vào bảng Orders CustomerId là ‘AFKLOL’.

    Nếu không được đặt khóa ngoại, bạn sẽ insert thành công vào bảng Orders, dẫn đến dữ liệu bị sai. Vì thực tế không có CustomerId nào là ‘AFKLOL’.
    Nếu đặt khóa ngoại, khi insert một CustomerId ảo, không có ở bảng Customers, SQL sẽ báo lỗi không cho phép insert.

    Chúng ta tạo khóa ngoại của một bảng thông qua ràng buộc FOREIGN KEY.

    Ví dụ: tạo một bảng Customers và Addresses có quan hệ với nhau.

    --Sử dụng database Master
USE master
--Kiểm tra database có tồn tại hay không mới xóa
IF EXISTS(select * from sys.databases WHERE name='Test')
DROP DATABASE Test
CREATE DATABASE Test 

GO 

USE Test;

CREATE TABLE Customers 
  ( 
     CustomerId int PRIMARY KEY IDENTITY( 1, 1 ), 
     Name       varchar(20) NOT NULL 
  ); 

CREATE TABLE Addresses 
  ( 
     AddressId  int PRIMARY KEY IDENTITY( 1, 1 ), 
     Street     varchar(50), 
     CustomerId int FOREIGN KEY REFERENCES Customers( CustomerId ) 
  );
    Copy

    F5 để chạy đoạn mã trên, bạn đã tạo ra bảng Customers và Addresses có quan hệ với nhau.
    Đoạn mã tạo mối quan hệ

    CustomerId int FOREIGN KEY REFERENCES Customers( CustomerId ));
    Copy

    Nhìn vào đoạn mã trên ta thấy, CustomerId trong bảng Addresses được đặt ràng buộc khóa ngoại, FOREIGN KEY, và ràng buộc này dẫn tới cột CustomerId là khóa chính của bảng Customers.

    Những lưu ý khi  tạo mối quan hệ giữa hai bảng:

    Dùng ràng buộc khóa chính (PRIMARY KEY) và khóa ngoại (FOREIGN KEY) để tạo mối quan hệ giữa hai bảng. Khóa ngoại của bảng này sẽ dẫn tới khóa chính của bảng kia.

    Sở dĩ phải dùng khóa chính vì khóa chính dùng để xác định chính xác record đó, mỗi record có một khóa chính khác nhau nên không thể gây ra nhầm lẫn.
