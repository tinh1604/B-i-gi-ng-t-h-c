Tính đóng gói trong OOP trong PHP

Tính đóng gói (encapsulation) "đóng gói" thuộc tính và phương thức của đối tượng (hoặc lớp) thông qua việc giới hạn quyền truy cập (hoặc thay đổi) giá trị của thuộc tính hoặc quyền gọi phương thức. Nói cách khác tính đóng gói cho phép kiểm soát quyền truy cập (và thay đổi) giá trị của thuộc tính hoặc quyền gọi phương thức của đối tượng (hoặc lớp) và đối tượng (hoặc lớp) con.
Trong OPP PHP việc đóng gói được thực hiện nhờ sử dụng các từ khoá public, private và protected:

  • public: Cho phép truy cập (và thay đổi giá trị) của thuộc tính và phương thức ở mọi phạm vi
  • private: Chỉ cho phép truy cập (hay thay đổi) giá trị của thuộc tính và phương thức ở phạm vi của đối tượng (hoặc lớp).
  • protected: Chỉ cho phép truy cập (hay thay đổi) giá trị của thuộc tính và phương thức ở phạm vi của đối tượng con (hoặc lớp con).

Để hiểu rõ hơn về tính chất này chúng ta sẽ xem một ví dụ sau.

<?php
class Person 
{
		// Phạm vi truy cập
		public  $fullName = "Nguyễn Hữu PHP";
		public $age ;
		 // Các phương thức
		public static function getFullName()
		{
			  return $this->fullName;
		}
		
} 
$per  = new Person();
echo $per->fullName = "Nguyễn Văn PHP <br/>";
echo $per->age = 24;
?>

Xem ví dụ

Như bạn đã thấy sau khi tôi khởi tạo một đối tượng với tên là $per thì tôi có thể thiết lập cũng như lấy giá trị của các thuộc tính của đối tương Person mà không gặp bất cứ vấn đề gì.

Tôi không muốn cho các các đối tượng khác và cả các lớp con kế thừa từ Person được phép truy cập vào một thuộc tính nào đó trong đối tượng Person giả sử là thuộc tính $Age. Lúc đó tôi sẽ phải sửa lại quyền truy cập của thuộc tính $age là:

private $age;

Và khi đó thì chúng ta không thể thiết lập giá trị cho thuộc tính $age theo cách thông thường như trên nữa. Việc thiết lập giá trị cho biến $age chỉ được thao tác từ bên trong đối tượng Person thì có một số cách sau:

1.Gán giá trị trong lúc khởi tạo đối tượng.

function __construct($age, $fullName)
	{
		$this->age = $age;
		$this->fullName = $fullName;
	}

2. Xây dựng hàm set giá trị cho thuộc tính. Trong hàm này quy định tuổi từ 1 đến 120

function setAge($age)
{
    if($age>=1 && $age<=120)
        $this->age = $age;
    else
        echo "Xin vui lòng nhập tuổi trong khoảng:1->120";
    
}

3. Và việc lấy giá trị của thuộc tính $age của đối tượng Person phải thông qua một hàm public function getAge()

// Lấy giá trị
function getAge(){
    return $this->age;
}

Tổng hợp

<?php
class Person 
{
		// Phạm vi truy cập
		private  $fullName ;
		private $age ;
		 // Các phương thức
		function __construct($age, $fullName)
		{
			$this->age = $age;
			$this->fullName = $fullName;
		}
		function setAge($age)
		{
			if($age>=1 && $age<=120)
				$this->age = $age;
			else
				echo "Xin vui lòng nhập tuổi trong khoảng:1->120";
			
		}
		function getAge()
		{
			return $this->age;
		}
		
		function setFullName($fullName)
		{
			$this->fullName = $fullName;
		}
		
		function getFullName()
		{
			  return $this->fullName;
		}
		
} 
$per  = new Person(24, "Nguyễn Hữu PHP");
echo $per->getFullName() ."<br/>";
echo $per->getage();

Xem ví dụ

Tính đóng gói (encapsulation) và che giấu thông tin (information hiding): Tính chất này không cho phép người sử dụng các đối tượng thay đổi trạng thái nội tại của một đối tượng. Chỉ có các phương thức nội tại của đối tượng cho phép thay đổi trạng thái của nó. Việc cho phép môi trường bên ngoài tác động lên các dữ liệu (thuộc tính) nội tại của một đối tượng theo cách nào là hoàn toàn tùy thuộc vào người viết mã. Đây là tính chất đảm bảo sự toàn vẹn của đối tượng.
